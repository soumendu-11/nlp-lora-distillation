# Cancer Abstract Classification — LoRA vs Knowledge Distillation

Comparative study of two parameter-efficient / teacher-guided training strategies for binary biomedical text classification (cancer vs non-cancer abstracts).

---

## Overview

| Approach | Description |
|---|---|
| **LoRA** | DistilBERT fine-tuned with Low-Rank Adaptation (r=4, α=32) — only ~0.93 % of parameters are trained |
| **Distillation** | DistilBERT student trained on soft probability labels generated by Azure GPT-4o — combines KL-divergence and cross-entropy loss |

Both models achieve **96 % test accuracy / 95.83 % F1** on 150 held-out biomedical abstracts.

---

## Dataset

1 000 biomedical PubMed abstracts (500 cancer, 500 non-cancer) stored as `.txt` files under `Dataset/`.

| Split | Samples |
|---|---|
| Train | 700 (70 %) |
| Validation | 150 (15 %) |
| Test | 150 (15 %) |

## Setup

```bash
# 1. Clone the repo
git clone <repo-url>
cd model_distillation

# 2. Create a virtual environment
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure Azure OpenAI credentials
cp .env.example .env             # then edit .env with your keys
```

### `.env` format

```
AZURE_API_KEY=<your-azure-openai-key>
AZURE_ENDPOINT=https://<your-resource>.openai.azure.com/
API_VERSION=2025-01-01-preview
DEPLOYMENT_NAME=gpt-4o
```

## Running the Pipeline

```bash
# Run all steps end-to-end (idempotent — skips completed steps)
python run_all.py

# Or run individual steps
python data_loader.py          # step 1: split data
python lora_training.py        # step 2: LoRA fine-tuning
python distillation_training.py  # step 3: knowledge distillation
```

After training, open `results.ipynb` to view plots and analysis:

```bash
jupyter notebook results.ipynb
```

## Key Results

| Metric | LoRA | Distillation |
|---|---|---|
| Accuracy | 96.00 % | 96.00 % |
| F1 | 95.83 % | 95.83 % |
| Precision | 100.00 % | 100.00 % |
| Recall | 92.00 % | 92.00 % |

### Why are the metrics identical?

The models differ on exactly **2 out of 150 test predictions** — in a perfectly symmetric swap — so aggregate metrics cancel out. See Section 11 of `results.ipynb` for a detailed breakdown.

**Key factors:**
- 80 % of GPT-4o soft labels were near-binary (P > 0.9 or P < 0.1), giving distillation no extra signal beyond hard labels
- LoRA reached 96 % validation accuracy in epoch 1 — the task is simple enough for both approaches to find the same decision boundary quickly
- 150 test samples is too small to detect the 2-prediction difference in aggregate metrics

## LoRA Architecture

- **Base model**: `distilbert-base-uncased`
- **LoRA rank (r)**: 4
- **LoRA alpha**: 32
- **Target modules**: `q_lin` (query projection in each attention layer)
- **Trainable params**: 628,994 / 67.6 M (0.93 %)

## Distillation Setup

- **Teacher**: Azure GPT-4o (zero-shot soft probability generation)
- **Student**: `distilbert-base-uncased` (all parameters trained)
- **Temperature (T)**: 4
- **Alpha (α)**: 0.7 (KL weight); 0.3 (cross-entropy weight)
- **Loss**: `L = α·T²·KL(student/T ‖ teacher) + (1−α)·CE(student, hard_label)`
- **Soft label cache**: `data_splits/soft_labels_cache.json` (avoids repeated API calls)
